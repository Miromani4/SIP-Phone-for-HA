# Release v1.1.9

## 🔍 Улучшенное логирование

### Что нового:
- **Детальное логирование подключения** - теперь можно отслеживать причины потери соединения
- **Счетчики попыток** - отслеживается количество потерь соединения и попыток регистрации
- **Временные метки** - логируется время каждого события для диагностики
- **Детализация ошибок** - в логах отображается тип исключения и полный контекст

### Новые логи:
```
# При старте:
INFO Starting SIP phone: user=<user>, server=<server>:<port>, realm=<realm>
INFO SIP socket bound to <local_ip>:<local_port>

# При потере соединения:
WARNING Connection lost (count: <N>) (exception: <exc>). 
        Local addr: <ip>:<port>, Server: <server>:<port>

# При регистрации:
INFO Starting registration loop: user=<user>, server=<server>:<port>
DEBUG Registration attempt #<N> to <server>:<port>
DEBUG Sending REGISTER to <server>:<port>, Call-ID: <id>, CSeq: <n>

# При таймаутах:
WARNING Registration timeout (5s) - no response from server. 
        Server: <server>:<port>, Call-ID: <id>, CSeq: <n>
```

### Использование для отладки:
1. Включите debug логирование в Home Assistant:
   ```yaml
   logger:
     logs:
       custom_components.sip_doorbell.phone: debug
   ```

2. Отслеживайте счетчики:
   - `Connection lost (count: X)` - поможет определить частоту разрывов
   - `Registration attempt #X` - покажет количество попыток регистрации

3. Проверяйте временные метки:
   - Помогут выявить паттерны в проблемах с подключением
